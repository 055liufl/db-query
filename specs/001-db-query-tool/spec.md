# 功能规格说明：数据库查询工具

**功能分支**：`001-db-query-tool`
**创建日期**：2026-03-29
**状态**：草稿
**输入**：用户描述 — 数据库查询工具，支持连接数据库、浏览元数据、直接 SQL 查询及自然语言生成 SQL

## 用户场景与测试 *(必填)*

### 用户故事 1 — 添加数据库连接并浏览元数据（优先级：P1）

用户在界面中输入一个数据库连接 URL，系统自动建立连接，抓取数据库中所有表
和视图的结构信息（字段名、类型、注释等），并将这些元数据以可浏览的列表形式
展示出来。同时，系统将元数据持久化缓存，下次打开同一连接时直接复用，无需
重新查询数据库。

**为何此优先级**：这是整个工具的基础入口，没有数据库连接和元数据，后续所有
查询功能均无法进行。它独立交付后即构成可用 MVP。

**独立测试**：输入有效的 PostgreSQL 连接 URL，点击"连接"，验证左侧面板展示
出该数据库的表和视图列表，以及每个表的字段信息。

**验收场景**：

1. **Given** 用户未添加任何连接，**When** 用户输入有效的数据库 URL 并点击
   "连接"，**Then** 系统在 30 秒内展示该数据库的所有表和视图列表，并显示
   每个表的字段名与字段类型。
2. **Given** 用户已添加并缓存了某个数据库连接，**When** 用户再次选择该连接，
   **Then** 系统立即从缓存加载元数据，无需重新查询数据库。
3. **Given** 用户输入了无效的连接 URL，**When** 用户点击"连接"，**Then**
   系统显示清晰的错误提示（如"连接失败：无法解析主机"），并保持当前界面
   可操作状态。

---

### 用户故事 2 — 直接编写并执行 SQL 查询（优先级：P2）

用户在 SQL 编辑器中输入一条 SQL 查询语句，系统对语句进行语法校验并确认其
为 SELECT 查询后，连接目标数据库执行查询，将结果以表格形式展示在页面下方。
若查询缺少 LIMIT 子句，系统自动补充 LIMIT 1000 以防止返回过大结果集。

**为何此优先级**：直接 SQL 查询是工具的核心价值，满足有 SQL 基础的用户的
即席查询需求，依赖 P1 的数据库连接功能。

**独立测试**：在已连接数据库的状态下，在 SQL 编辑器输入 `SELECT * FROM
users`，点击"执行"，验证结果以表格展示（最多 1000 行）。

**验收场景**：

1. **Given** 已成功连接数据库，**When** 用户输入合法的 SELECT 语句并执行，
   **Then** 系统在 10 秒内以表格形式展示查询结果，列名与数据库字段一致。
2. **Given** 用户输入的 SELECT 语句不含 LIMIT 子句，**When** 执行查询，
   **Then** 系统自动为该语句追加 `LIMIT 1000`，并在界面上提示用户已应用
   默认行数限制。
3. **Given** 用户输入非 SELECT 语句（如 `DROP TABLE users`），**When** 点击
   执行，**Then** 系统在提交前拒绝执行并显示错误提示"仅允许执行 SELECT
   查询"，数据库不受任何影响。
4. **Given** 用户输入语法错误的 SQL，**When** 点击执行，**Then** 系统显示
   具体的语法错误位置与描述，不向数据库发送请求。

---

### 用户故事 3 — 自然语言生成 SQL 查询（优先级：P3）

用户用中文或英文自然语言描述想要查询的内容（例如"查询最近 7 天注册的用户
数量"），系统将当前数据库的表结构信息作为上下文传给 AI，AI 生成对应的 SQL
语句，填入 SQL 编辑器中供用户确认或修改后再执行。

**为何此优先级**：自然语言生成 SQL 降低了使用门槛，但依赖 P1（元数据）和
P2（SQL 执行）功能，且需要外部 AI 服务，优先级低于核心查询功能。

**独立测试**：在已连接数据库的状态下，在自然语言输入框输入"查询所有用户的
邮箱"，点击"生成 SQL"，验证 SQL 编辑器中出现对应的 SELECT 语句，用户可
编辑后执行。

**验收场景**：

1. **Given** 已连接数据库且元数据已加载，**When** 用户输入自然语言描述并点击
   "生成 SQL"，**Then** 系统在 15 秒内在 SQL 编辑器中填入 AI 生成的 SELECT
   语句，用户可直接查看、编辑并执行。
2. **Given** AI 生成了 SQL，**When** 用户点击"执行"，**Then** 生成的 SQL
   同样经过语法校验（仅允许 SELECT），校验通过后才提交执行。
3. **Given** AI 无法根据描述生成有意义的 SQL，**When** 系统收到 AI 响应，
   **Then** 系统提示用户"无法理解该查询描述，请尝试更具体的表述"，SQL
   编辑器内容不变。

---

### 边界情况

- 数据库连接超时（超过 30 秒未响应）时如何处理？
- 元数据中包含极大数量的表（如 500+）时，列表的展示与性能是否受影响？
- 查询结果为空时，表格区域如何提示用户（空状态提示）？
- SQL 语句包含多条语句（用 `;` 分隔）时如何处理？
- 数据库连接在查询过程中意外断开时如何处理？
- 自然语言描述中涉及不存在的表名时，AI 生成结果是否给出提示？

## 需求 *(必填)*

### 功能需求

- **FR-001**：系统 MUST 允许用户输入数据库连接 URL 并保存至本地存储。
- **FR-002**：系统 MUST 在连接成功后自动获取目标数据库的所有表和视图的结构
  信息（表名、视图名、字段名、字段类型）。
- **FR-003**：系统 MUST 将数据库元数据持久化缓存到本地，同一连接 URL 的元
  数据可复用，无需每次重新查询。
- **FR-004**：系统 MUST 在界面中以可浏览的形式展示表和视图列表及字段信息。
- **FR-005**：系统 MUST 提供 SQL 编辑器，用户可在其中输入 SQL 语句。
- **FR-006**：系统 MUST 对所有输入的 SQL 进行语法解析，拒绝语法不正确的语
  句并显示具体错误信息。
- **FR-007**：系统 MUST 拒绝执行非 SELECT 语句的 SQL，并显示明确的拒绝原因。
- **FR-008**：系统 MUST 对不含 LIMIT 子句的 SELECT 查询自动追加 `LIMIT 1000`，
  并在界面告知用户。
- **FR-009**：系统 MUST 将查询结果以表格形式展示，包含列名和对应数据。
- **FR-010**：系统 MUST 提供自然语言输入框，将用户描述与数据库元数据一起发
  送给 AI，并将返回的 SQL 填入 SQL 编辑器。
- **FR-011**：系统 MUST 在 AI 生成的 SQL 进入编辑器后，用户确认（点击执行）
  才提交查询，不得自动执行。

### 核心数据实体

- **数据库连接（DatabaseConnection）**：唯一标识、连接 URL、显示名称、最后
  连接时间、连接状态。
- **数据库元数据（DatabaseMetadata）**：所属连接 ID、表/视图名称、字段列表
  （字段名、类型、是否可空、注释），以 JSON 格式缓存。
- **查询记录（QueryExecution）**：所属连接 ID、SQL 原始文本、实际执行的 SQL
  （含自动追加的 LIMIT）、执行状态（成功/失败）、错误信息、结果行数。

## 成功标准 *(必填)*

### 可衡量的成功指标

- **SC-001**：用户从输入连接 URL 到看到完整的表和视图列表，全程不超过 30 秒。
- **SC-002**：SQL 语法校验结果在用户点击执行后 2 秒内反馈，非法语句立即被
  拦截，不向数据库发送任何请求。
- **SC-003**：查询结果表格完整展示所有返回列，在结果不超过 1000 行时不出现
  明显的界面卡顿或渲染延迟。
- **SC-004**：自然语言生成的 SQL 填入编辑器后，用户无需修改即可直接执行的
  比例达到 70% 以上（基于常见的单表查询场景）。
- **SC-005**：所有错误场景（连接失败、SQL 非法、AI 无响应）均有用户可理解
  的中文提示，不展示技术堆栈错误信息。

## 假设

- 目标数据库类型为 PostgreSQL，元数据通过查询 PostgreSQL 系统表获取；其他数
  据库类型（MySQL、SQLite 等）不在本期范围内。
- 本工具为内部开发或数据分析人员使用，用户具备基本的数据库概念（了解表、
  视图、SQL 查询），无需面向完全非技术用户设计。
- 不实现任何身份认证，所有用户均可直接访问所有功能。
- 同时在线的用户数量较少（≤10 人），不针对高并发场景进行优化。
- AI 生成 SQL 依赖外部 LLM 服务，假设该服务可用；若服务不可用，用户仍可
  手动编写 SQL。
- 运行环境为 Win10 + Docker，前后端均通过 Docker 容器部署，开发者本地启动
  即可使用。
- 移动端适配不在本期范围内，仅支持桌面浏览器。
